Phase 2: High Priority Security & Stability RECOMMENDED

Important security hardening and stability improvements for production.

  • 2.1 Add Content Security Policy Headers ~1-2 hours
    Implement strict CSP headers to prevent XSS attacks.
    // next.config.js async headers() { return [{ source: '/(.*)', headers: [ { key: 'Content-Security-Policy', value: "default-src 'self'; script-src 'self' 'unsafe-eval' 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; font-src 'self'; connect-src 'self' https://*.googleapis.com https://*.firebaseio.com;" }, { key: 'X-Frame-Options', value: 'DENY' }, { key: 'X-Content-Type-Options', value: 'nosniff' }, { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' } ] }] }
    next.config.js
  • 2.2 Implement Request Timeout Handling ~1 hour
    Add timeout handling to all external API calls (Google Ads, Meta, TikTok, Snap).
    // lib/api-client.ts const controller = new AbortController(); const timeout = setTimeout(() => controller.abort(), 30000); try { const response = await fetch(url, { signal: controller.signal, ...options }); } finally { clearTimeout(timeout); }
    lib/google-ads.ts, lib/meta-ads.ts, lib/tiktok-ads.ts, lib/snap-ads.ts
  • 2.3 Add Circuit Breaker Pattern ~2-3 hours
    Implement circuit breaker for external API calls to handle failures gracefully.
    npm install cockatiel // lib/circuit-breaker.ts import { CircuitBreakerPolicy, ConsecutiveBreaker } from 'cockatiel'; export const apiCircuitBreaker = new CircuitBreakerPolicy({ breaker: new ConsecutiveBreaker(5), // Open after 5 failures halfOpenAfter: 30000 // Try again after 30s });
    lib/circuit-breaker.ts
  • 2.4 Enable Firestore Security Rules ~1-2 hours
    Deploy production Firestore security rules with proper user isolation.
    // firestore.rules rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { match /users/{userId} { allow read, write: if request.auth != null && request.auth.uid == userId; } match /tokens/{tokenId} { allow read, write: if request.auth != null && resource.data.userId == request.auth.uid; } match /reports/{reportId} { allow read: if request.auth != null && resource.data.userId == request.auth.uid; allow write: if request.auth != null && request.resource.data.userId == request.auth.uid; } } }
    firestore.rules
  • 2.5 Set Up Error Monitoring ~1-2 hours
    Integrate error monitoring service (Sentry recommended) for production visibility.
    npm install @sentry/nextjs // sentry.client.config.ts import * as Sentry from '@sentry/nextjs'; Sentry.init({ dsn: process.env.SENTRY_DSN, tracesSampleRate: 0.1, environment: process.env.NODE_ENV });
    sentry.client.config.ts, sentry.server.config.ts

Phase 4: Production Deployment FINAL STEPS

Steps to deploy and launch in production environment.

  • 4.1 Set Up Production Environment Variables ~1 hour
    Configure all production environment variables securely.
    Variable Description Required
    NEXTAUTH_URL Production URL (https://yourdomain.com) Yes
    NEXTAUTH_SECRET 32+ character random string Yes
    TOKEN_ENCRYPTION_KEY 64 hex characters (32 bytes) Yes
    FIREBASE_* Firebase project credentials Yes
    GOOGLE_* Google OAuth & Ads API credentials Yes
    META_* Meta/Facebook API credentials Optional
    TIKTOK_* TikTok API credentials Optional
    SNAP_* Snapchat API credentials Optional
    REDIS_URL Redis connection string Yes
    SENTRY_DSN Sentry error tracking DSN Recommended
  • 4.2 Configure Firebase Production Project ~1 hour
    Set up separate Firebase project for production with security rules.
  • 4.3 Set Up OAuth Redirect URIs ~30 minutes
    Add production URLs to OAuth app configurations for all platforms.
    Important: Update redirect URIs in Google Cloud Console, Meta Developer Portal, TikTok Developer Portal, and Snap Business Manager.
  • 4.4 Build & Test Docker Image ~30 minutes
    Build production Docker image and test locally.
    # Build image docker build -t daily-reporting:latest . # Run locally with production config docker-compose -f docker-compose.yml up -d # Test health endpoint curl http://localhost:3030/api/health # Run smoke tests npm run test:e2e
  • 4.5 Deploy to Production Infrastructure ~1-2 hours
    Deploy to your chosen platform (Vercel, AWS, GCP, etc.).
    # Option A: Vercel (Recommended for Next.js) vercel --prod # Option B: Docker to Cloud Run gcloud run deploy daily-reporting \ --image gcr.io/PROJECT_ID/daily-reporting \ --platform managed \ --region us-central1 \ --allow-unauthenticated # Option C: Docker to AWS ECS aws ecs update-service --cluster prod --service daily-reporting --force-new-deployment
  • 4.6 Configure SSL/TLS Certificate ~30 minutes
    Ensure HTTPS is properly configured with valid SSL certificate.
  • 4.7 Set Up Monitoring & Alerts ~1 hour
    Configure uptime monitoring and alerting for critical issues.
    Recommended Tools: UptimeRobot or Pingdom for uptime, Sentry for errors, CloudWatch/Stackdriver for infrastructure.
  • 4.8 Verify Production Deployment ~1 hour
    Run through complete user flows in production.
