2. Backend API Audit

Critical Security Issues

Critical GET Logout Endpoint Allows CSRF

Route: /api/auth/logout

The logout endpoint accepts GET requests, which can be triggered by embedding an image tag or link, causing unintended logouts.

Fix: Change to POST-only endpoint with CSRF token validation.

Critical Open Redirect in Auth Callback

Route: /api/auth/callback

The returnUrl parameter is not validated, allowing attackers to redirect users to malicious sites after authentication.

Fix: Validate returnUrl against whitelist of allowed domains/paths.

Critical PDF Generation Not Implemented

Route: /api/reports/generate

The report generation endpoint stores HTML as binary instead of generating actual PDF files. Users receive corrupted files.

Fix: Implement proper PDF generation using Puppeteer or similar library.

Critical In-Memory Rate Limiting

Affected: All rate-limited endpoints

Rate limiting uses in-memory storage which doesn't work with multiple server instances and resets on restart.

Fix: Use Redis or database-backed rate limiting.

Performance Issues

  • Sequential API Calls: Platform sync executes 6 API calls sequentially instead of in parallel
  • No Caching: BigQuery results not cached, causing repeated expensive queries
  • Synchronous Operations: PDF generation and data sync block the request thread
  • Missing Indexes: Firestore queries lack composite indexes for multi-field queries

3. Frontend Audit

Critical Accessibility Issues

Critical Toggle Switches Missing ARIA

Component: Settings Page

Custom toggle switches lack role="switch" and aria-checked attributes, making them inaccessible to screen readers.

High Native confirm() Dialogs

Affected: Profile deletion, Account disconnect

Using JavaScript confirm() which is not accessible and cannot be styled. Use custom accessible modal instead.

High Charts Not Keyboard Accessible

Component: Dashboard Charts (Recharts)

Interactive charts cannot be navigated with keyboard, excluding users who cannot use a mouse.

Security Issues

High Delete Account Not Protected

Page: Settings

Account deletion button has no confirmation dialog, no password re-entry, and no grace period. Accidental clicks cause permanent data loss.

High Firebase Errors Exposed

Pages: Login, Signup

Raw Firebase authentication error codes displayed to users, potentially revealing account existence information.

4. Recommendations
